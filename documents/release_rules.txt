How to release GRASS 5 binaries:

0. Check version string:

   src/CMD/VERSION

   Check if no M/C conflicts are there before tagging.

   Update the PDF-docs on server. Run html/Gmakefile (activate postscipt-
   target, convert to PDF in dist.$ARCH/documents/postscript, upload to
   server)


1. Build GRASS:
 
 o wipe out binaries, (re-)compile everything,

 o compile PostgreSQL modules:
     gmake5 src.garden/grass.postgresql

 o compile G3D tools
     gmake5 src.contrib/GMSL/g3d

 o compile NVIZ *without* PostgreSQL support
   for distribution:
     configure --enable-postgres=no
     make pre-compile
     gmake5 src.contrib/GMSL/NVIZ2.2

 o check, if libgdal is in place ($GISBASE/lib/)
   ftp://gdal.velocet.ca/pub/outgoing/libgdal-linux-grass.tar.gz
   (ldd libgdal.1.1.so:
    ftp://gdal.velocet.ca/pub/outgoing/libstdc++-libc6.2-2.so.3)

 o compile s.vol.rst from extra directory (not distributed with sources)


2. Package it with

   make bindist


3. Publish it on web site:
 - grass5x_y_bin.tar.gz
 - grass5install.sh
