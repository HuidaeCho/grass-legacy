This directory contains the WINDOWS/Cygwin binary code of GRASS 5.0 in zipped format.

This is the first version with experimental graphical map output.

REQUIREMENTS:
There are three major requirements to run GRASS 5 on WINDOWS: The "CygWin" which provides a UNIX compliant environment and the Xserver which allows to use GRASS' graphical output. Finally the "IPC" package is needed to allow GRASS communication with graphical driver.

a) You must have installed Cygwin for WINDOWS from http://sources.redhat.com/cygwin/download.html
Get "Cygwin" package there (setup.exe). Cygnus is required to run UNIX compliant applications on WINDOWS.

b) Additionally get the XWindows precompiled client (Xfree) from http://sources.redhat.com/cygwin/xfree/. Get the packages: DLLs, fonts, Xterm, Xwin, Prog. (Xfree install guide)

c) Building IPC for Grass5:
You will need the Inter-process communication (IPC) code: http://www.neuro.gatech.edu/users/cwilson/cygutils/V1.1/cygipc/ These binaries are included in the grass5 zip file; however you will need the .a archive to build on your own.
See See compile_grass_with_wincygnus.txt for more details.

Installation of GRASS 5 Windows binaries:
It is a relative root path so someone can unzip it to their D: drive if necessary. Depending upon where you have your cygwin installed, it could go under that root directory: - store GRASS 5 zip into the main directory of CygWin
(e.g. /programs/cygwin)
- unzip grass5-xxxxxxxxx.zip (replace xxx with current version number) using your favourite "unzip" software - change to: /programs/cygwin/usr/local/grass5/ and double click "StartGrass"

Starting Grass5:
The package includes a StartGrass script in the /usr/local/grass5. This sets the PATH, start the ipc daemon, and spawns an xterm window. From within that window run 'grass5'.

Compiling the GRASS 5 locally - CVS Grass5:
