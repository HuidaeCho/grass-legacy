DPG Library

This library was built in support of the GRASS 5.0 Vector API. But it was designed to provide several useful reusable functions. It started out as a collection of independant libraries. My concept of a library is a re-usable module of independant code. This, when taken to an extreme (like I do with most things), can turn out to be a very small piece of code which does not deserve the title of library. So I have gathered these all together placed them into a single library. This is not unreasonable as many of the modules are built on top of other modules.

All names in the global name space begin with either the prefix dpg or tavl in any combination of upper and lower case letters. DPG are my initials and I couldn't come up with anything more creative than that. Tavl is there because that is the prefix of a library for an AVL tree writen by Bert C. Hughes. It was so well designed and written almost in the same way I was designing my libraries that I didn't want to desecrate it by changing its prefix.

The modules available in the DPG libarary are:

  • Object Cache
  • Object Storage
  • String management
  • Age manager
  • Debug
  • Bitmap
  • Dynamic Array
  • Dictionary
  • Linked List Memory Manager
  • Linked List Memory Manager Manager
  • Integer Tree Tavl Front end
  • TAVL Tree
  • Stdio
  • Xdr Front end
  • Malloc Front end
Note I am not real proud of a few of these. I was very rushed in building this library and there are a few things that should be done differently. Unlike the Vector API, this library did not have a design document before coding began. None the less, a few of them are quite powerful and have a lot of potential.
