GRASS 5.0 Vector API: Topology Read

Topology Read

Status: Alpha
This module has been designed but not thoroughly reviewed.
These routines provide access to the data contained in the map. This functionally was available in previous versions only by digging into the data structures themselves. In 5.0 the data structures are no longer accessable and all requests for information must go though this interface. All information about connectivity of map features is accessable.

GETTING POINTS INFO 

gvStatus gvGetLinePoints (gvMapP map, gvPidP line, gvPointsP points)

This routine will populate the initialized gvPointsP structure with the
vertices of the specified line.

INPUT
    gvMapP map
    gvPidP line		Line of interest

OUTPUT
    gvPointsP points	The user provides an initialized gvPointsP 
			structure which is filled with the line vertices.
RETURN
    gvStatus
	GV_OK		Success
	GV_FAILED	Error
	GV_MEMORY_ERR	Out of memory
index 
gvStatus gvGetAreaPoints (gvMapP map, gvPidP area, gvPointsP points)

This routine will populate the initialized gvPointsP structure with the
vertices of the specified area.  The points will be returned in order
clockwise around the area.  The starting point is arbitrarily chosen.

INPUT
    gvMapP map
    gvPidP area		Area of interest

OUTPUT
    gvPointsP points	The user provides an initialized gvPointsP 
			structure which is filled with the area vertices.
RETURN
    gvStatus
	GV_OK		Success
	GV_FAILED	Error
	GV_MEMORY_ERR	Out of memory
index 
gvStatus gvGetIslePoints (gvMapP map, gvPidP isle, gvPointsP points)

This routine will populate the initialized gvPointsP structure with the
vertices of the specified island.  The points will be returned in order
counter-clockwise around the island.  The starting point is arbitrarily chosen.

INPUT
    gvMapP map
    gvPidP isle		Isle of interest

OUTPUT
    gvPointsP points	The user provides an initialized gvPointsP 
			structure which is filled with the isle vertices.
RETURN
    gvStatus
	GV_OK		Success
	GV_FAILED	Error
	GV_MEMORY_ERR	Out of memory



General Map Info



The following routines provide access to information about a specific
map.  The first set of routines cover general information and the latter
routine provide access to individual information about each primitive 
feature.
index 
int gvPrimCount (gvMapP map, gvAllType type)

Returns count of active primitive types in map 

INPUT
    gvMapP map
    gvAlltype type   One of:
			GV_PT_LINE    All Lines
			GV_PT_AREA    All Areas
			GV_PT_NODE    All Nodes
			GV_PT_ISLE    All Isles
			GV_PT_LLINE   Lines of type LINE
			GV_PT_ALINE   Lines of type AREA
			GV_PT_NLINE   Lines of type NEAT
			GV_PT_CNODE   Connecting Nodes
			GV_PT_ENODE   Entity Nodes

OUTPUT
    gvPointsP points	The user provides an initialized gvPointsP 
			structure which is filled with the isle vertices.
RETURN
    int		0 and positive:    Number of specified elements
		Negative:	   Bad inputs

NOTE
    The only primitives are LINE, AREA, NODE.  The others are subsets 
    of these.  Isles are there for convenience.
index 
double gvGetSnapThresh (gvMapP map)	

This routine returns the current value of the snapping threshold.  This
is the distance in map units in which a new line will automatically snap
to an existing node.

INPUT
    gvMapP map

OUTPUT
    None

RETURN
    double 	Snapping threshold
index 
double gvGetMapThresh (gvMapP map)	

This routine returns the current value of the map threshold.  This 
value is by default determined based on map scale and an arbitrary
factor based on assumptions about map accuracy.  By default the snapping
threshold and pruning threshold are derived from this value.
    

INPUT
    gvMapP map

OUTPUT
    None 

RETURN
    double 	Map threshold
index

A map can be in one of at least three states.

1. Partially complete topology
2. Fully complete topology
3. Fully complete toplogy with inconsistancies
These inconsistancies can be things such as polygons with more than one area marker within them, polygons with no area markers within them (which is not an error), incompleted polygons (in LINE/AREA mode).

gvBoolean gvIsTopoComplete (gvMapP map)

This function will tell whether the topology information is complete
or not.  If it is not, that means edits have been done recently and
a topology build is required to complete the topologic information.

INPUT
    gvMapP map

OUTPUT
    None 

RETURN
    gvBoolean 	  GV_TRUE   Topology is complete
              	  GV_FALSE  Topology is not complete
index 
gvOpenMode gvGetRwMode (gvMapP map)

This function will tell which mode the map has been opened in.  The
mode is one of READ, WRITE or UPDATE.  In read mode, NO modification
of the map can be done.   The primary difference between write and
update modes, is that update works with a temporary copy of the map
and only updates the permanent copy on request.

INPUT
    gvMapP map

OUTPUT
    None 

RETURN
    gvOpenMode		One of:         GV_OPEN_READ
					GV_OPEN_UPDATE
index 
gvTopoMode gvGetTopoMode (gvMapP map)

This function returns whether the topology for this map is in the old
GRASS 4.0 format or the new GRASS 5.0 format.

INPUT
    gvMapP map

OUTPUT
    None 

RETURN
    gvTopoMode		One of:         GV_TOPO_OLD
					GV_TOPO_NEW
index

gvPrimType gvGetLineType (gvMapP map, gvPidP pid)

This function returns the type of the line.  A line can be classified
as type  AREA [boundary], LINE, or NEAT.  A Neat line is used to represent
the outer extent of a map and can be used for various types of functions 
such as automatic snapping and line breaking, disolving of borders on 
merge, as well as being used to form polygons in normal use.

INPUT
    gvMapP map
    gvPidP pid

OUTPUT
    None 

RETURN
    gvPrimType	    One of:	GV_PT_LINE 
				GV_PT_AREA 
				GV_PT_NEAT 

NOTE   
    In the new 5.0 topology design, the LINE/AREA distinction is 
    eliminated.  Thus the only types that will be returned in 
    in GV_TOPO_NEW maps are  GV_PT_AREA and GV_PT_NEAT.

gvPrimType gvGetNodeType (gvMapP map, gvPidP pid)

This function returns the type of the node.  A line can be classified
as Connecting or Entity.   A connecting node is a node that exists
at the end of a line segment.  It is used for joining 1 or more lines
together at a point.  An entity node must NOT intersect with any 
map line or node feature and is used to represent zero dimensional
spatial data.

INPUT
    gvMapP map
    gvPidP pid

OUTPUT
    None 

RETURN
    gvPrimType	    One of:	GV_PT_CNODE 
				GV_PT_ENODE

ISLE STRUCTURE

NOTE: An island is NOT a proper geographic primitive feature. It is provided simply for the capability to represent area holes in their entirety. Every island will in effect contain one or more valid area features. Although for consistancy of the interface, islands can be represented with a gvPidP data type, this should never be used for attributing or indexing into a database.

int gvGetPidDirection (gvPidP pid)

This function returns the direction of a pid, 
have a direction and Node 1 is the starting node.

INPUT
    gvPidP pid

OUTPUT
    None 

RETURN
    int		One of:
			-1	Reverse, Start
			 0	Not set
			 1	Forward, End

NOTE
    When refering to nodes, reference Start and End, when refering
    to ordering of lines, reference Reverse and Forward.  Reverse
    meaning that the ordering of the points need to be reversed.

    This function could be replaced by a function that queries 
    the appropriate information from the Node, Area and Isle objects.
