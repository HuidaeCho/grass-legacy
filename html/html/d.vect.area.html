d.vect.area

NAME

d.area - Draws filled vector areas to the GRASS display
(GRASS Display/Vector Program)

SYNOPSIS

d.vect.area [-r] map=name [color=name|R:G:B] [legend=filename]

Flags:

-r
"Randomly" fill all areas by cycling through standard GRASS colors

Parameters:

map
Name of existing vector map to be displayed
color
Color desired for filling polygons. May be specified as a standard grass color name, or as an RGB triplet (e.g. 80:198:255).
Default: white
legend
Name of a legend file specifying what color to use for each category (see below).
Default: none

DESCRIPTION

Use d.vect.area to draw labelled areas from a vector map to the GRASS display. Using, the "-r" flag, the program will cycle through the standard GRASS named colors, overriding any color argument. If a legend file is specified, it will be used. Otherwise, the color used for filling areas is as specified.

The color argument may be any of the standard named GRASS colors, or and RGB (red, green, blue) triplet. An RGB triplet must have the form <num>:<num>:<num> where num is between 0 and 255.

More control can be had by creating a legend file. At the time of this writing, GRASS does not have any management functions for specifying how vectors should be drawn, so this file must be created by "hand". The easiest way to create such a file, is to copy the category file for the vector map, and edit it to contain color specifications. The format of the legend file is category red green blue, with one entry per line. If a category for a map is not included in this file, it will not be drawn unless there is a special "default" entry with a category of -1.

Example Legend File
-1 255 255 255
1 210 198 168
5 168 198 220

In the above, all labelled areas will be drawn in white (255, 255, 255), except those labelled as categories 1 or 5. If the "default" entry had been omitted, then only areas labelled 1 or 5 would have been drawn.

NOTES

The d.vect.area module does not draw boundary lines. It only flood fills areas.

It is expected that the legend specification aspect of this module will change when an infrastructure for managing vector drawing styles is implemented.

BUGS

It might be considered a bug, that malformed legend file entries are silently dropped. Only if no entries could be parsed, will an error be raised.
